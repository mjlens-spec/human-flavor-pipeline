## 脱敏流程(多轮,记录在案)

1. **Bitable 81 条记录**:机械脱敏(按客户名称字段定向替换)→ 3 路独立 agent 并行人工复核(逐条比对全部客户名单及可能的子品牌 / 竞品 / 关联方,共修正 45/81 条)→ 自动化交叉核查(品牌根词 0 残留)。
2. **PDF/PPTX 提炼语料(30 份文档)**:提炼时脱敏 → 发现原始提炼结果混入 AI 助手内部推理痕迹与"脱敏说明自曝品牌名"两类失败模式 → 机械剥离推理痕迹 → 各文件派独立 agent 做不依赖清单的通读复核,专门排查「脱敏说明自曝」与「XX 可以保留的例外辩解」两种模式 → 修正多处(含清单方式完全无法预知的品牌名,如案例小标题直接写死品牌简称)→ 全量交叉检查确认 0 残留。
3. **最终把关**:AI 自动化脱敏与 agent 复核完成后,用户本人逐字通读全部原文,亲自确认可公开范围 —— 期间判定其中一份材料敏感度不易把握,决定保留私有。这一步由人工亲自过目,不是仅凭 AI 的过程描述放行。
4. **脱敏原则**:客户 / 品牌 / 产品线名一律替换为行业类别描述;GMV / ROI / 播放量 / 涨粉数等可辨识具体项目效果的数字一律模糊化或删除;内部员工 / 客户联系人真实姓名一律删除;人群特征数字、市场价格带、方法论阈值定义(如"行业前 10%")视为品类常识予以保留;媒体平台名(今日头条 / 抖音 / 小红书等)与头条易自研工具产品名(巨量云图 / MarKOL 等)不算敏感信息,予以保留。
5. **教训**:清单式核查(拿已知客户名单去 grep)会漏掉清单之外的品牌(如「客户 logo 墙」里从未出现在案例库里的服务客户名、案例小标题里直接写死的品牌简称、第三方数据机构署名)。真正可靠的方法是不依赖清单的通读复核,清单只能作为复核完成后的最终交叉验证,不能替代通读;而无论 AI 复核多少轮,涉及真实第三方商业关系的公开发布决定,最终仍需人工亲自逐字确认。

## 与 human-flavor-pipeline 主体的关系

这批语料是独立的参考资料,不是 `patterns/`(操作层规则)的一部分。`wording-glossary.md` 里的词汇本身不含任何客户信息,是风险最低、最适合未来优先考虑并入公开 `catalog/whitelists.md` 的部分;`company-playbook.md` / `cross-industry-methodology.md` 的「可复用结构」小节沉淀了大量文案写作/提案骨架手法,可作为 `exemplars.md` 的潜在补充素材,是否并入留待后续决定。
