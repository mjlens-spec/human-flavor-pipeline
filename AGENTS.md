# human-flavor-pipeline · 中文去 AI 味集大成流水线

这是一条 polish / transform 管线,不是生成器。它只改已经存在的文本,把它变成读起来像人认真写的终稿。它不替你产生缺失的东西 —— 观察、采访、记忆、判断要靠人,这条管线做的是底层文字清洁与人味注入。

设计目标不是骗过 AI 检测器(脆弱的军备竞赛),而是让你自己的 AI 辅助稿子确实读起来像人写的。检测分下降是结果,不是目的。

> **跨工具**:本文件是 Claude Code 入口;Codex 走根目录 `AGENTS.md`(由 `scripts/build-agents.sh` 从本文件同源生成)。`patterns/` 与 `references/` 数据层两个工具共享,按需读取。改规程改本文件,再重跑脚本。

## 八条底线原则(贯穿全流程)

1. **检测先于改写** —— 先看懂再动手,诊断和编辑分两步,不边读边改。
2. **事实神圣**(最高优先)—— 数字、人名、机构、报价、引用、代码开改前锁进保护区,收尾逐项核对,任何漂移即拦截回退。
3. **门检优先** —— 先判人写还是 AI 写。人写的只清格式,不重写。
4. **语体阶梯,至多漂移一级** —— 输出语体相对输入最多偏移一格(阶梯见下),跨级要用户确认。
5. **保留人味** —— 终稿至少留一处「毛边」(具体判断 / 具体感受 / 圈内说法),且通过「AI 不敢写」测试;但毛边只能从原文里捞,**伪造毛边等于违反原则 2**。
6. **分级 ＋ 语体门控** —— 同一条规则在不同语体下结果不同,按渠道与语体门控,不一刀切。
7. **全程透明** —— 交付的是带双评分的 diff 报告,不是黑箱换稿。
8. **有界迭代** —— 用强度档位控制改动幅度,未达标且档位允许时最多回改 N 遍(默认 N=2),不无限催降。

## 调用模式 ＋ 场景门

**场景门(先判这是什么场合)**:
- **快速 / 聊天场景**(用户随手贴一句要改、对话里顺带去味)→ 默认轻档,直接给改后版,报告精简成一两行。
- **正式交付场景**(策划、提案、客户方案、公众号长文、特稿)→ 默认先 detect 体检,再问要不要全量改;改完出完整打磨报告。

**两种模式**:
- **detect(只读体检)** —— 只跑阶段 0–1,输出 `AI 味分 0–100`(越低越好)＋ `人味质量分 0–50`(越高越好)＋ 热区 ＋ 主导 pattern 类型,**一个字都不改**。适合特稿这种「先要体检」的场景。
- **full(全量改写)** —— 跑完整六段,输出终稿 ＋ 打磨报告。

用户没指定时,按场景门定默认;除非已明确说「直接改 / 帮我改好」,正式场景也先体检再改。

## 强度档位(由 AI 味分决定,可被用户覆盖)

| 档位 | AI 味分 | 改动幅度 |
|------|---------|----------|
| 轻 | 0–15 | 仅 in-place 词汇替换,不动结构 |
| 中 | 16–35 | bounded 删改:删句 / 合并先列候选清单,低风险直接删 |
| 深 | 36–100 | structural 重构:自由删句、合并、重排、改写结构 |

双评分细则见 `references/scoring.md`。

## 语体阶梯与声口

**九种语体**(从输入推断或由渠道指定):社交 / 口语 · 内容 / 自媒体 · 商务 / 职场 · 书面 / 一般 · 叙事非虚构 / 特稿 · 品牌广告 / 文案 · 学术 / 科技 · 公文 / 法律 · 应试作文。

**语体阶梯**(正式度从高到低),输出至多偏移一格:

```
公文 / 法律 ＞ 学术 / 科技 ＞ 商务 / 职场·对外 ＞ 书面 / 一般·自媒体 ＞ 内部口语·朋友圈
```

**声口(voice profile)** 与语体正交,决定句式与锋芒:直白 / 温和 / 犀利 / 技术 / 叙事。每种声口是一组「句长норм ＋ 用词倾向 ＋ 结构许可」的捆绑,见 `patterns/voice-profiles.md`。没指定时按渠道默认。

## 六阶段执行流程

### 0 · 门检 / 锁定 / 定调
开改前做四件事:
1. **判来源** —— 人写还是 AI 写 / AI 辅助?**人写的就停手**,只清格式(标点、空格)。判断写成一行 `【门检】` 输出。判定线索见 `references/self-audit.md`。
2. **锁保护区** —— 扫出所有数字、百分比、金额、报价、日期、人名、机构名、产品 / 车型 / 版本号、专有名词、直接引语、代码、报错日志,列成「不可改清单」。
3. **定语体 ＋ 阶梯锚点** —— 推断输入语体,记下阶梯位置(输出至多偏移一格)。
4. **定渠道 ＋ 声口** —— 按 `patterns/channel-presets.md` 选渠道预设,按 `patterns/voice-profiles.md` 选声口;用户没给就声明假设。

### 1 · 诊断打分(只读)
按 `patterns/banned-words.md` 的 A–I 模式分组和 `references/scoring.md` 跑检测,输出:
- `AI 味分 0–100` ＋ `人味质量分 0–50`
- **热区**:点名哪几段最重(段号 ＋ 主导问题)
- **主导 pattern 类型**:对应 A–I 哪几组

**此阶段不改任何字。** detect 模式到此结束并交付体检报告。

### 2 · 剥离(先结构,后词汇,再标点)
顺序是要点 —— 结构信号比词汇信号强,先动结构收益更高。

**① 先结构**:
- 打破三段式排比 → 改 2 项(取最强两项)或 4 项,破对称
- 删开场烘托句(「随着……的不断发展」「在……的大背景下」)
- 删结尾感召段(「综上所述」「相信……」式收束)
- 去机械路标词(「首先 / 其次 / 最后」)
- 结论前置:把埋在段尾的判断提到段首
- **空句检测**:逐句问「删掉它,信息有没有损失?」,无损失的空句(空泛判断、填充强调)标记为候选删除

**② 后词汇**:按三级词典(词典文件 B 组)处理 —— Tier 1 无条件改、Tier 2 同段超密度才动、Tier 3 仅全文密度异常时标记。

**③ 六大润色动作**(qu-ai-wei §6 精华,主动施加):
1. **动词强化** —— 把名词化弱表达换成有力动词(「进行优化」→「优化」「做出决策」→「定下来」)
2. **节奏重塑** —— 长短句交错(详见阶段 3)
3. **填充词删除** —— 删无信息填充(「其实」「应该说」「某种程度上」)
4. **抽象落具体** —— 把空泛判断落到具体,但只用原文已有的具体信息
5. **语序归位** —— 把翻译腔 / AI 腔的别扭语序改回中文自然语序
6. **语体匹配** —— 让用词与目标语体、声口一致

**④ 最后标点**:按渠道预设规整破折号、全角 / 半角、引号、中英文间半角空格。

全程受语体子集约束:特稿 / 品牌广告语体保护铺垫节奏与低密度破折号。

### 3 · 注入人味
去味之后要防「换成另一种机器腔」。三个动作:
- **节奏(burstiness)** —— 显式制造长短句落差:有意安排一句 6 字内短句紧接一句 25 字以上长句,打破句长均一。
- **风格锚点 ＋ 声口** —— 若 `patterns/style-anchors.md` 有用户真文样本,用它做 few-shot 对齐用户声口;否则按选定 voice profile。
- **留毛边** —— 保至少一处具体判断或具体感受,从原文已有信息里捞,不能编。

### 4 · 校验(事实优先,两遍读)
**第一遍只查事实**,第二遍才查 AI 腔残余 —— 顺序不能反,否则会因删 AI 腔而误删信息。
1. **事实保真(第一遍)** —— 拿阶段 0 的保护区清单逐项比对终稿,任何数字 / 实体被改动或丢失即拦截回退。
2. **AI 残余 ＋ 反过度净化(第二遍)** —— 复算双评分;查残留套话;同时确认没把人味连同 AI 味一起削平。
3. **AI 不敢写测试 ＋ 整篇五问自检** —— 见 `references/self-audit.md`,终稿须至少含一句「AI 出于安全 / 正确 / 讨好默认不会写」的具体带态度句子,并通过五问。

**高风险场景**(客户方案 / 特稿,或走「深」档)派一个独立子 agent 做对抗式复核(Claude Code 用 Task 工具;Codex 用其子 agent / 多轮独立复核机制):「事实有没有被改动?读起来还像不像 AI?」,主流程不自评自己。

### 5 · 报告交付
套渠道排版(引号、空格、分段),输出终稿 ＋ 打磨报告。若复打分仍超阈值且档位允许,回阶段 2 做有界回改(默认最多 2 遍),否则交付并标注残留项。

## 输出格式(打磨报告)

```
【门检】<人写 / AI 写 / AI 辅助;采用语体·阶梯位·渠道·声口>
【评分】AI 味 XX→YY(越低越好) | 人味 AA→BB(满 50,越高越好)
【保护区核对】<逐项确认数字 / 实体原样保留;有改动则逐条说明>
【主要改动】(按 A–I 分组,带原文→改后的真实引用,不是举例)
  · 结构(C/G/H):<删三段排比 / 删开场烘托 ……>
  · 词汇(B/D):<赋能→带动 / 综上所述→(删) ……>
  · 润色(六动作):<进行优化→优化 / 删填充「其实」……>
  · 标点(I):<破折号→逗号 / 补半角空格 ……>
【保留的毛边】<指出终稿里刻意留下的人味,及它为何过「AI 不敢写」测试>
【残留项】<没改或不敢改的地方,及原因>
```

快速 / 聊天场景可把报告压成一行:`AI 味 XX→YY,改了 N 处(结构 / 词汇),事实无损`。

## 边界与禁区

- **不编造事实** —— 输出里所有具体信息必须能在输入里找到来源。
- **不从零生成** —— 没内容可改时直接说明。
- **人写优先** —— 门检判为人写就只清格式。
- **不过度净化** —— 宁可少改也别洗成无菌的另一种 AI 腔。
- **只简体中文** —— 不处理英文正文与繁体。
