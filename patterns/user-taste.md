# 个人禁忌层(user taste)

**最高优先级**的个人规则层 —— 凌驾于通用词典之上。同一个词,通用层放行、个人层禁用时,**以个人层为准**。这一层装的是「这个用户的语言洁癖」,从用户的成文偏好和手动改稿习惯里沉淀。

> 使用者按自己的禁忌增删本文件。下面是默认装填(基于本项目用户的写作偏好与实战),换人用请替换。

---

## 一、个人禁用(高于通用 Tier 1)

用户明确点名要避的「江湖气」表述,见到即改,不看语体:

- 集中火力、临门一脚、打透、打爆(作动词比喻时)
- 挑大梁、主战场、卡位 / 卡口、稳住盘面、冲刺(作比喻时)
- 「边界给死、空白留给我们」这类口号式表述
- 否定先行模板(`不是 X,而是 Y`、`不是 X,但 Y`、`不需要 X,要 Y`)默认避免,但**策略分野例外**:当 X 与 Y 是可量化、不可对调的策略选项(例如「不做日常投放,做一个事件」且两者预算 / 目标差异明确)时,单次放行作合法对比;同段扎堆 ≥2 次才改成肯定陈述。与通用层 `negation_first_cluster` 密度门控对齐(见 `banned-words.md`),个人层不再一刀切硬禁
- 小标题不以否定句起手,先写清主体是什么、任务是什么、要做什么;确需划边界时,把限制放在肯定结论之后

替换方向:换成具体动作或直白书面词。否定先行句先改成肯定判断,再补原因或边界。直接引语和必须逐字保留的事实仍服从事实保护区。

## 二、个人正面习惯(从手动改稿沉淀,主动遵守)

用户反复在手动修改里坚持的做法,改写时主动对齐:

- **数据必带口径** —— 增长口径 / 构成口径、来源(灵犀 / 云图 / 艾媒…)随数字标注;无来源的数据标「待确认」,绝不替它编出处。
- **拿不准就标「待确认」** —— 建议项、未对齐项显式列「待确认」清单,不含糊带过。
- **小标题写成结论,不写名词** —— 「银饰:主动需求上行,三套系对上三个增量点」优于「银饰市场分析」。
- **直角引号「」**,嵌套『』;不用弯引号 ""。
- **汉字与英文 / 数字间留一个半角空格**。
- **结论前置**,把「因为 / 但问题在于」显式写出。
- **乙方 deck 结构是专业,不是 AI 味** —— 战略角色 / KFS 分配 / 节点日历 / 人群分层该保留;只去「假大空愿景 / 无数据承诺」。见 `channel-presets.md` 的乙方腔边界。
- **递进用「→」,不用「首先 / 其次 / 最后」** —— 「从 A → 进入 B → 进入 C」式路径递进,每条尽量等长、12 字内;序号连词留给真正需要编号的清单。
- **人群描写用「她们 / 他们」＋行为细节,不用状态词断言** —— 写「她们住在城市的缝隙里,想让房间有一点自己的气味」,不写「她们孤独焦虑」;用可观测行为 / 场景承载情绪。
- **callout 框功能分区** —— 📌＝数据 / 执行补层,💡＝人群情绪 / 创意洞察;两类不混用,读者靠图标判断阅读优先级。
- **收尾不升华** —— 章节 / 方案结束于具体动作或待确认清单(「下一步:① 回填站内数据 ② 文案过法务」),不写「相信品牌会迎来更好的明天」式空话。
- **「N 件事」归纳体** —— 先报集合数(「三件事的状态」「两张牌」「四个判断」),再逐条展开,帮读者在细节前先锚定框架;执行要干净,不加虚词。

## 三、个人保留(别误删)

用户场景里合法、不要当 AI 腔清掉的行业术语:种草、破圈、货架、铺量、私域、KOC / KOL、CPE / ROI、KFS、品效合一、节气营销、挂车、店播。

## 优先级

冲突仲裁:**事实神圣 > 个人禁忌层(本文件) > 受众 / 文体 / 笔调 > 载体格式 > 通用词典**。
