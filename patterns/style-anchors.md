# 风格锚点(few-shot)

经验结论:**给范文比堆规则更有效**。在阶段 3 注入人味时,与其只靠禁用词表约束,不如塞 1–2 段目标风格的真实文字做 few-shot,让输出对齐一个具体的人的笔调,而不是对齐一个通用「人类」。

这一点被 shuorenhua 的 `.writer/` 参考文章库和 OUBIGFA 的 good-writing 模式共同验证。

---

## 怎么用

1. 在下面「锚点库」里贴入你自己的真文段落,或你想模仿的目标范文(你欣赏的优质媒体 / 作者、你过往发表过的稿子等;贴时记得脱敏,别留客户名)。
2. 调用本 skill 时,阶段 3 会读取这里的样本,把终稿的用词、句式、节奏向样本靠拢。
3. 不同渠道可以存不同锚点。没有锚点时,退回 `channel-presets.md` 的渠道默认语体。

**注意**:锚点只影响「笔调 / 风格」,不影响事实。锚点里的具体信息不会被搬进你的稿子(那会违反事实原则)。

---

## 锚点库(在此粘贴你的真文)

### 锚点 · 通用 / 默认
> (在此粘贴 1–2 段你最满意的、最像你本人的文字)

### 锚点 · 对外提案 / 客户方案
> (在此粘贴一段你过往的高质量提案文字)

### 锚点 · 长报道 / 特稿
> (在此粘贴一段你欣赏的特稿 / 你自己的报道段落)

### 锚点 · 公众号
> (在此粘贴一段目标公众号风格文字)

### 锚点 · 主播 / 口播
> (本地粘贴 1–2 段已经播出或你确认满意的逐字稿片段;公开仓库不要提交个人经历、节目母题、固定收尾、客户信息或未公开素材)

## 已提炼笔调参数(脱敏 · 暂无真稿时的 fallback 锚)

> 来源:2026-06 从本项目用户真实方案 / 策略稿提炼,已脱敏(不含品牌 / 客户 / 报价)。真稿请在上方锚点库本地补、勿提交公开 repo;真稿优先级高于本表。这张参数表是没有真稿时也能用的笔调锚。

| 维度 | 参数 |
|------|------|
| 句长 | 策略判断句 15–25 字,金句 / 结论句 8–14 字,数据支撑句可到 35 字;长短交替不拉平 |
| 连接 | 因果外显(「所以」「但问题在于」「这说明」「含义:」);递进用「→」;意合只在情绪描写段用 |
| 标点 | 直角引号「」;中英 / 数字间留空格;冒号或「→」也常用来作展开 / 递进;破折号「——」不禁用,适度使用即可 |
| 用词 | 具体动词驱动(接住 / 安放 / 落进 / 翻译);自造概念标签;行业词与抒情句分区不混 |
| 节奏 | 短段砸判断;数据 / 逻辑段中等长度;callout 分层(📌＝数据,💡＝人群情绪);结尾不升华 |
| 人称 | 洞察层「她们」描述用户,策略层「我们」说动作,文案金句无人称;不用「您 / 大家」 |

标志手法:结论砸到最短独立成句(「价格亲民,状态很贵。」);递进用「→」每条 12 字内;逻辑链「所以」显式接结论;「她们」人群画像用行为细节替代状态词;「待确认」作为正式交付项显式留白,不假装已知。

## 提取风格的维度(供阶段 3 参考)

读锚点时关注这些,而不是照抄内容:
- 句子的平均长度与长短句落差
- 常用的连接方式(显式连词 vs 意合)
- 标点习惯(逗号密度、是否用破折号、问句频率)
- 用词偏好(书面 vs 口语、成语密度、是否爱用具体名词)
- 段落节奏(是否爱用短段、是否有停顿与留白)
- 人称与语气(第一人称介入度、克制 vs 热情)
- 口播特征(承重句是否短准、连接组织是否自然、软垫词 / 反问 / 自我修正是否承担听觉功能)
