# A–H 表达模式 ＋ I 风险 ＋ J 风格偏好

数据层,与流程解耦。A–H 用于表达模式诊断,I 只记录事实 / 来源 / 格式风险,J 记录商业表达与个人风格。I/J 不并入 AI 味分。词汇强度按 Tier 1/2/3 分级,但任何一级都要结合上下文,不做裸字符串定罪。

替换建议是方向,不是死规定;具体替换服从语体、声口与上下文。

## 两层结构:操作层 ＋ 深查层

- **本文件是操作层** —— A–H 表达模式 ＋ I/J 独立清单 ＋ 三级强度,日常改写按这里快速判断。
- **深查层在 `catalog/`**(并自 qu-ai-wei,MIT,见 `CREDITS.md`)—— 当一条模式拿不准是否触发、或要看原文 / 改后对照与语体限定时,翻深查层:
  - `catalog/patterns.md` —— 51 条逐条定义(原文 / 改后对照、语体限定、跨规则 interlock)
  - `catalog/platform-patterns.md` —— H 组平台 / 文体(公众号 / 小红书 / B 站脚本等)
  - `catalog/whitelists.md` —— 完整白名单(体育 / 医学 / 技术 / 护肤成分 / 明星昵称等豁免)
  - `catalog/punctuation.md` · `catalog/syntax.md` —— 标点指纹与句法反模式细则
  - `catalog/brand-voice.md` —— 品牌广告语体下每条规则的特殊判定
  - `precision-rules.json` —— 供 production 规则与 precision 契约共用的阈值、上下文和例外

## 词汇强度分级(贯穿 B / D 组)

- **Tier 1 强信号** —— 先核对文体、功能与语义;确认是空转表达后再改 / 删
- **Tier 2 中信号** —— 同段超密度(≥2 次)才动
- **Tier 3 弱信号** —— 仅全文密度异常时标记,**默认不强删**

## A · 内容注水

为凑字数 / 显得充实的铺陈、同义反复、把一句话拆成三句说。

- 展开写法:同一意思换三种说法重复
- 「为了……,我们需要……,只有这样才能……」式空转因果链
- 把结论拆成「背景 + 解释 + 重申」三段说一遍

处理:压缩成一句,留信息删铺陈。

## B · AI 高频词(按 Tier 分级)

**Tier 1**:璀璨、熠熠生辉、画卷、华章、新篇章、谱写、绽放、赋予新的内涵、浓墨重彩、淋漓尽致
**Tier 2**:然而、因此、此外、与此同时、进一步而言、在此基础上、基于以上、综合来看
**Tier 3**:重要、关键、核心、显著、有效、积极、深入、全面、持续、不断、有力、扎实

替换:然而→但是 / 可是;因此→所以;此外→另外 / 还有;Tier 1 多数直接删或换具体描述。

## C · 修辞腔

- 三段式排比(恰好三项、整齐对称)→ 改 2 项或 4 项,破对称
- 否定肯定并列:不仅……更是……、不是……而是……、既……又……(密集时)
- 设问自答:「这意味着什么?这意味着……」→ 直接陈述
- 感召收束:「让我们……」「相信……」
- 假坦诚开场:「说实话 / 坦白讲 / 我本来不想说」后面没有真实代价、局限或具体经历
- 格言公式 / 伪洞察枢纽:把普通判断压成可换题复用的「真正的 X 是 Y」「归根结底……」

处理:保留点睛一处,其余改直陈。

## D · 客服腔 / PR 腔(按 Tier 分级)

**Tier 1**:赋能、助力、打造、护航、抓手、闭环、底层逻辑、深度赋能、全方位、多维度、全面升级、生态闭环、协同推进、价值赋能、深度探索
**Tier 1(机器礼貌)**:希望本文对您有所帮助、让我们一起、让我们来看看、欢迎持续关注、感谢您的阅读

替换:赋能→帮 / 带动 / 让……能;打造→做 / 建 / 写;抓手→办法 / 切口;闭环→走通;机器礼貌句一律删。

## E · 填充词(无信息)

其实、应该说、某种程度上、从某种意义上、不可否认、坦率地说、众所周知、毋庸置疑、值得注意的是、需要指出的是、值得一提的是、不言而喻、显而易见

处理:删前缀直接说事;扎堆时尤其要清。口播声口下,「其实 / 就是 / 我觉得 / 那 / 对吧」等若承担呼吸、转场、反问或自我修正功能,不按裸字符串处理;只有无信息堆叠时才删。对应阶段 2 的「空句检测」与「填充词删除」。

## F · 翻译腔

- 别扭语序:「正是因为……才使得……」「这是一个关于……的故事」
- 滥用「之一」:「最重要的原因之一」→ 直接说原因
- 被动滥用:「被认为是」「被广泛应用于」(非学术语体)
- 「对于……来说」「就……而言」过密

处理:语序归位成中文自然表达(对应六大润色动作之「语序归位」)。

## G · 结构与节奏

| 反模式 | 识别 | 处理 |
|--------|------|------|
| 句长均一 | 连续多句近等长 | 按信息量重组,不套固定长短句公式 |
| 段落等大 | 每段字数相近 | 让段落有起伏 |
| 机械分段 | 每段都「首先 / 其次」 | 去路标词,自然过渡 |
| 数字列表强迫症 | 通篇 ①②③ / 1. 2. 3. | 高密度时改行文段落 |
| 同义词循环替换 | 为避重复强行换词致词汇漂移 | 还原最准的词,允许重复 |
| 连续短句造戏剧感 | 三句以上极短句只负责制造气势 | 合并成承载因果或动作的完整句;品牌文案与口播节奏除外 |
| 标题后复述标题 | 首段换词重说标题,没有新信息 | 直接进入事实、判断或动作 |
| 变更叙事腔 | 正文夹入「这次我把 X 改成 Y」等编辑过程 | 删除编辑日志;变更说明 / changelog 语体除外 |

## H · 平台套路

- 开场烘托:「随着……的不断发展」「在……的大背景下」「在当今……的时代」「近年来,随着……」
- 假权威引用:「研究表明」「业内人士认为」「据相关资料显示」(无具体出处时)
- 标题党式过渡句
- 助手续写尾巴:「如果你愿意,我还可以……」「需要的话我可以继续……」混入可直接发布的正文

处理:开场烘托整句删;无出处的假引用删或标注「待确认」,绝不替它编出处。

## I · 事实 / 来源与格式风险(不计 AI 味)

- 编造的数据 / 出处 / 引语 / 人名(**违反原则 2,最严重**)→ 拦截,标注「待确认」
- markdown 源码出现在不支持 Markdown 的目标载体
- emoji 残留(非小红书 / 朋友圈语体)
- 全角 / 半角混用、中英文间缺空格
- UTM / referrer 追踪参数、来源卡片、搜索结果面板残留
- 模板占位符(`XX 路`、`[产品名]`、`{{变量}}`):提示补全或核对,不证明作者身份

处理:事实风险先核验;格式风险按目标载体修复。UTM、占位符与 Markdown 只进入风险清单,不抬高 AI 味分。

## J · 江湖气 / 互联网黑话

中文商业写作的高频风格问题,不等于 AI 来源证据。只有比喻空转、语义含混或命中用户个人禁忌时才改。**用户个人禁忌优先于本组**,见 `user-taste.md`。

**军事 / 体育比喻**:集中火力、临门一脚、打透、挑大梁、主战场、卡位 / 卡口、稳住盘面、冲刺、弯道超车、降维打击、引爆
**互联网黑话(阿里腔)**:赋能、抓手、闭环、打通、沉淀、对齐、拉通、全域、势能、链路、抢占心智、组合拳

替换速查:
| 黑话 | 替换方向 |
|------|----------|
| 赋能 | 帮 X 实现 Y / 让……能 |
| 抓手 | 切入点 / 执行路径 |
| 闭环 | 走通从 X 到 Y 的链路 |
| 打通 | 打破 X 和 Y 之间的隔断 / 接通 |
| 集中火力 | 集中预算投在 X |
| 临门一脚 | 最后一步是 X |
| 打透 | 说清 / 做到位 |
| 挑大梁 | 当主力 / 承担主要增长 |
| 稳住盘面 | 稳住基本盘 / 守住存量 |
| 引爆 | 带动单月销量翻 X 倍 / 引发 X 万次讨论 |
| 全域 / 全方位 / 多维度 | 删,说清具体是哪几个 |

> 注意词边界,别误伤:如「主**打通**勤」里的「打通」是「主打+通勤」,不是黑话。命中要回上下文核验(见 `references/scoring.md`)。

> 合法业务术语也放行:「需求拆解」「项目落地」「承接流量」「接住咨询」「系统撑不住峰值」「闭环控制」「转化链路」在语义准确、受众熟悉时保留。判据不是词是否出现,而是删掉后是否损失一个明确的业务关系。

## 检测下限(floor)

精度调校不得把检测清零。以下铁 slop 无论语境一律照抓(对应 `precision-rules.json` 的 floor 规则),作为「精度优先」的反向护栏,防止 skill 退化成什么都不敢改:

- 开场烘托:随着……的发展 / 在当今……时代 / 在……的大背景下(`cliche_opener`)
- 收尾套话:综上所述 / 由此可见 / 总而言之(`summary_cliche`)
- 空强调:值得注意的是 / 毋庸置疑 / 不言而喻 / 众所周知(`empty_emphasis`)
- PR 黑话:赋能 / 抓手 / 护航(`pr_empowerment`)

floor 与召回由 `tests/fixtures/19-recall-floor.jsonl` 的 `pos_*` 用例守住:测试既证「干净文本不误判」,也证「真 slop 必被抓」。这是对「只测不误报、奖励少改」的纠偏。

## 否定先行密度门控

「不是 X 而是 Y / 不是 X 但 Y」是合法对比修辞,**单次放行,扎堆才罚**:同一段出现 ≥2 次(`negation_first_cluster`)才判为公式化套路;品牌文案语体豁免。这取代了早期一刀切的硬禁,详见 `user-taste.md`。

## 词表校准

AI 味词汇随主力模型迭代演化(例如新版模型的「油腻味」与旧版不同)。词表与 floor 标「最后校准模型 / 日期」,见 `precision-rules.json` 的 `calibration` 字段;换主力模型或每季度复核一次,新味补进 J 组与 floor。

## 白名单(不处理)

以下场景里的「疑似词」是正常用法,放行:
- 体育、军事、医学、控制论等领域术语(如「闭环」在控制 / 医学语境)
- 技术缩写与专有名词
- 直接引语内的原话(属保护区,不改)
- 人名、品牌、产品昵称、语义准确的行业术语
- 学术 / 科技语体里的被动句与客观陈述(不算 AI 腔)
- 用户在 `style-anchors.md` 里明确保留的个人用语
