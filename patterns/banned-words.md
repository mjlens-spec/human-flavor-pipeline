# 三级禁用词典 ＋ 结构反模式

数据层,与流程解耦 —— 改词表不必动 `SKILL.md`。分级思路借鉴 shuorenhua、humanizer-skill 的 tiered dictionary:按信号强度分三级,避免一刀切误伤正常表达。

替换建议是方向,不是死规定;具体替换要服从语体与上下文。

---

## Tier 1 · 强信号,无条件改 / 删

这些词出现基本等于暴露 AI,见到就处理。

### 商业套话 / PR 腔
赋能、助力、打造、护航、抓手、闭环、底层逻辑、深度赋能、全方位、多维度、全面升级、生态闭环、协同推进、落盘、深度探索、价值赋能

替换方向:赋能→帮 / 带动 / 让……能;打造→做 / 建 / 写;抓手→办法 / 切口;闭环→走通 / 形成循环。多数情况直接拆成具体动作。

### 起承转合套话
综上所述、由此可见、总而言之、不难发现、显而易见、众所周知、毋庸置疑、不言而喻、可想而知

替换方向:多数可直接删;要保留逻辑就改成「所以」「说白了」。

### 强调空话
值得注意的是、需要指出的是、不可忽视的是、值得一提的是、必须强调的是

替换方向:删掉前缀,直接说那件事;或改「注意」「重点是」。

### 开场烘托模板
随着……的不断发展、在……的大背景下、在当今……的时代、近年来,随着……、在数字化浪潮下

替换方向:整句删,直接进入主题或事实。

### 机器礼貌 / 收尾
希望本文对您有所帮助、让我们一起、让我们来看看、欢迎持续关注、感谢您的阅读

替换方向:删。

### 辞藻堆砌(滥用义)
璀璨、熠熠生辉、画卷、华章、新篇章、谱写、绽放、赋予新的内涵、浓墨重彩

替换方向:删形容词,留事实;或换成具体描述。

## Tier 2 · 中信号,同段超密度(≥2 次)才处理

单独出现可能正常,扎堆才是 AI 腔。

- 逻辑连词堆叠:然而、因此、此外、与此同时、进一步而言、在此基础上、基于以上、综合来看
  - 替换:然而→但是 / 可是;因此→所以;此外→另外 / 还有
- 机械分段:首先 / 其次 / 再次 / 最后(论述体里的强对称版)
- 否定肯定并列:不仅……更是……、不是……而是……、既……又……(密集使用时)
- 程度虚化:在一定程度上、从某种程度上、相对来说、某种意义上
- 总结口头禅:总的来说、简而言之、概括而言

处理原则:保留一两个做真过渡,删掉其余充数的。

---

## Tier 3 · 弱信号,仅全文频率异常时标记

正常写作也会用,只有密度明显偏高才提示,**默认只标记不强删**。

重要、关键、核心、显著、有效、积极、深入、全面、持续、不断、有力、扎实

处理原则:在报告里提示「『重要 / 关键』全文出现 N 次,偏高」,交用户定夺。

## 结构反模式(比词汇信号更强,优先处理)

| 反模式 | 识别 | 处理 |
|--------|------|------|
| 三段式排比 | 恰好三项、整齐对称的并列 | 改 2 项(取最强两项)或 4 项,破对称 |
| 开场烘托句 | 首段用大背景铺垫才进正题 | 删,直接进事实 / 冲突 / 数据 |
| 结尾感召段 | 末段「综上 / 相信 / 让我们」式收束 | 删或改成克制的直接结论 |
| 数字列表强迫症 | 每段都 ①②③ 或 1. 2. 3. | 高密度时改自然行文段落 |
| 否定肯定并列滥用 | 通篇「不是 X 而是 Y」 | 保留点睛一处,其余改直陈 |
| 成语 / 四字格堆砌 | 一句里多个工整成语 | 减密度,换具体描述 |
| 句长均一 | 连续多句近等长 | 注入长短句落差 |
| 同义词循环替换 | 为避重复强行换词致词汇漂移 | 还原最准确的那个词,允许重复 |

## 白名单(不处理)

以下场景里的「疑似词」是正常用法,放行:
- 体育、军事、医学等领域术语(如「闭环」在控制论 / 医学语境)
- 技术缩写与专有名词
- 直接引语内的原话(属保护区,不改)
- 人名、品牌、产品昵称
- 用户在 `style-anchors.md` 里明确保留的个人用语
