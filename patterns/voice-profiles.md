# 声口(voice profiles)

借 humanizer-skill 的 voice-profile 思路:**声口不是词汇皮肤,而是一组「节奏倾向 ＋ 用词倾向 ＋ 结构许可」的捆绑**。只换词不换结构,会得到另一种机械腔;声口要从句式层面调整。

声口与语体正交 —— 语体定正式度,声口定句式与锋芒。同一份客户方案可以是「直白」声口也可以是「犀利」声口。没指定时按受众、文体和原文推断。

---

## 六种声口

### 直白(Blunt)
- 句长:短句为主,一句一个意思
- 用词:口语化动词,少形容词
- 结构:结论砸在最前面,不绕弯,不铺垫
- 适合:内部文档、快速结论、决策层汇报
- 反面:别写成电报体,该交代的逻辑要交代

### 温和(Warm)
- 句长:中等,偶有第二人称
- 用词:语气柔,但不肉麻、不卖萌
- 结构:先共情再给观点,过渡自然
- 适合:对外沟通、用户向内容、培训材料
- 反面:警惕滑回「希望对您有所帮助」式客服腔(D 组)

### 犀利(Sharp)
- 句长:长短交错,关键判断用短句砸
- 用词:敢用有态度的词,带锋芒
- 结构:亮观点 → 给证据 → 下判断,不和稀泥
- 适合:评论、策略提案的洞察段、行业分析
- 反面:锋芒要有证据撑,不能空喊;判断要能追溯

### 技术(Technical)
- 句长:中等,信息密度高
- 用词:术语精确,被动句正常(不算 AI 腔)
- 结构:定义 → 机制 → 边界条件,逻辑外显
- 适合:技术文档、产品说明、科技分析
- 反面:精确 ≠ 啰嗦,删掉解释性废话

### 叙事(Narrative)
- 句长:长短句落差大,有留白
- 用词:具体名词、场景细节
- 结构:场景 / 事实切入,保护铺垫节奏,结尾克制
- 适合:特稿、人物、商业深度报道、公众号长文
- 反面:细节要服务于解释人 / 关系 / 机制,不为氛围而氛围

### 口播(Spoken)
- 句长:承重句短而准,连接组织可以累加、回环和轻微自我修正
- 用词:第一人称、口头转场、反问和少量软垫可以保留;专业英文或圈内词按作者稳定声口处理
- 结构:具体现场 / 亲历案例 → 一个判断 → 证据或比方层层垫 → 回到判断或画面收束
- 适合:播客逐字稿、视频口播、演讲稿、直播讲稿
- 反面:不得凭空补亲历、数字、口头禅、私货彩蛋或固定 coda;也别把所有「其实 / 就是 / 我觉得」洗成书面句

---

## 用法

阶段 0 选定声口后,阶段 2 的「语体匹配」和阶段 3 的「节奏 / 锚点」都按该声口执行。若用户在 `style-anchors.md` 贴了真文,真文优先于声口预设 —— 真文是最具体的声口锚点。
