## 初稿

AI 生成中文已经比较成熟,主流工具都能吃得下中文输入、也能输出像样的中文。下面按三类整理。

**1. 中文文本(写作、翻译、对话)**

擅长文章、诗歌、邮件、文案这类文本。

- 通义千问:阿里的模型,中文处理能力不错。
- ChatGPT / Gemini:国际主流模型,当下的中文也处理得相当可用。

**2. 带中文的图片生成**

早期 AI 绘图的中文乱码问题,现在可以直出清晰的中文汉字。

- 即梦 AI:国产模型,能直接生成带准确中文的图片,适合海报、文案设计。
- 豆包 App:文生图功能也支持中文海报。

- SeedDream / SeedEdit:视频生成和编辑模型,支持中文交互和排版效果。

## 自审

### AI 不敢写测试(底线)

问:终稿里有没有一句 AI 不敢写的?

**答:没有。**

原文是一份"AI 工具清单 + 使用建议",没有任何:个人具体使用经历、带情绪的偏好判断、承认记不清 / 不确定、自嘲、别人不会知道的具体细节。整篇是对象化的工具说明。

按 v0.5.5「🚫 事实发明禁令」和 v0.6.0「🚫 硬约束:craft ≠ 虚构授权」,**不得为了通过"AI 不敢写测试"凭空发明毛边**(不能编"我上周用文心一言写生日贺卡,它把我妈名字写成繁体"之类)。正确做法是**如实报告失败**:原文缺乏毛边,无法在不虚构的前提下通过底线测试。这比一个"读起来有人味、内容是假"的终稿更诚实。

**处理方式:** 终稿保留"中性散文"手感,不补发明细节。用户看打磨报告和本自审就能判断:要不要自己补一句真实使用体验。

### 整篇 craft 自检(合格线)

1. **用词精准** — 随机挑 5 处:`比较成熟` `吃得下` `像样的` `跟上` `往专业方向拉一拉`。前四处都承载具体含义(不可无损删),最后一处"拉一拉"是轻度口语 filler,但承担"效果有限 / 不一定"的弱修饰功能,保留。**通过。**
2. **节奏变化** — 朗读全篇,最短 9 字(`大致就是这些`),最长约 45 字(首段半句);没有"连 3 句 20-30 字完整主谓宾"的机关枪段。**通过。**
3. **动词驱动** — 全篇动词:吃得下 / 输出 / 整理 / 擅长 / 支持 / 响应 / 处理 / 直出 / 生成 / 处理 / 做 / 接 / 出 / 支持 / 设定 / 写出 / 拉 / 试。弱动词 `打造 / 构建 / 形成 / 发挥` 为 0 处,`进行` 0 处。**通过。**
4. **具体替抽象(原文已有)** — 原文本身没有可挖的具体细节(没有数字、没有具体产品版本、没有使用场景 ROI)。终稿没发明,也没让抽象漏过去(抽象的"多个领域"被"三类:文本 / 图像 / 视频 PPT"替代,抽象的"最佳选择"被"可以先试 X / 国际模型也可用"替代)。**通过(在"原文已有"边界内)。**
5. **声音一致** — 全篇是一个"懂一点 AI 工具的朋友在跟你说人话"的声口,没有前半公文、后半朋友圈的跨档问题。**通过。**

### 残留痕迹(至少两条)

1. **底线测试未通过** — 整篇仍是"干净的中性散文",没有一句 AI 不敢写的。原文限制所致,非改写者偷懒。
2. **`大致就是这些` 稍微"装在收句"** — 是一种补偿性口语化,意图填"没有毛边"的空,本质仍是中性散文的收尾手势,不算真人的毛边。
3. **工具名音译的括注彻底删除后,`即梦 AI` / `豆包 App` 这种命名残留带一点 AI 味** — 但这是**原文信息**,不能删(违反信息完整性),只能保留。
