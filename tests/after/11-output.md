## 自审

### AI 不敢写测试

- ✅ 「这次组织调整到底改了什么,原文没讲」—— 直接承认原文是空的,AI 默认会顺着原文继续写拔高句,不敢指出"这段话什么都没说"。
- ✅ 「通篇在说'这不是 X,而是更深刻的 Y',但 X 和 Y 都是抽象桶词,落不到一件真实的事」—— 带判断口吻,不是 AI 默认的平衡得体语气。

**关于事实发明禁令:** 原文通篇没有任何具体事实(团队名 / 时间 / 汇报结构 / 指标),改写不补造"从下周一起 X 团队向 Y 汇报""合并了 3 个部门"这类细节。终稿只能把原文的空转抽象层剥掉,然后**诚实报告原文给不出落地内容**,让用户自己补真实事实再来改。这符合 SKILL.md「遇到原文没有毛边时的正确做法」第 3 条 —— 告知用户原文不够,不替用户编。

### 整篇 craft 自检

1. **用词精准** —— 终稿里没保留「组织范式 / 系统性升级 / 组织势能 / 结构性跃迁 / 新坐标系 / 全新篇章」任何一个抽象桶词。✅
2. **节奏变化** —— 短句「原文没讲」和长句「至少得告诉他们:谁向谁汇报、谁负责哪块...」交替,没有均质机关枪。✅
3. **动词驱动** —— 「改 / 讲 / 说 / 落 / 告诉 / 汇报 / 负责 / 做 / 生效」全是真动作,没有「激活 / 重塑 / 跃迁」这类抽象万能动词。✅
4. **具体替抽象(原文已有)** —— 原文没给任何具体细节,所以没东西可换;代替方案是**如实说明原文的空**,并列出"如果要让读者有用"需要补什么信息。✅
