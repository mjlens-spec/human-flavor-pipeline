## 初稿

大语言模型在长上下文（long context）场景下的 latency 表现，是当前工业界与学术界共同关注的性能瓶颈之一。随着 context window 的持续扩展，模型在进行推理时的 throughput 会呈现显著的非线性下降，这一现象与注意力机制的平方复杂性密切相关。

然而，单纯地压缩 KV cache 或者对 attention 进行稀疏化，往往会在准确性上付出代价。近期的工作尝试通过分层缓存与选择性重算的组合策略，在不显著降低质量的前提下，对长序列推理进行加速。

此外，不同任务对 latency 的敏感性存在差异。对于代码补全这类交互式场景，首 token 延迟比总吞吐更关键；而对于离线的文档摘要，总 throughput 的优化收益更明显。这种任务驱动的优化思路，正在重塑推理框架的设计原则。
