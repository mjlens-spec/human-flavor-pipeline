# 自检协议

阶段 4 校验的细则。集合改写必要性门检、原有人味核对、整篇五问与独立复核。

---

## 改写必要性门检(阶段 0 用)

门检只判断文本是否需要改,不判断作者身份。倾向低必要性的线索:
- 有具体的、可能不完美的个人细节(真实地名、真实时间、私人记忆)
- 有不对称的判断、犹豫、自我修正
- 句长自然起伏,偶有口语、省略
- 有圈内黑话、行业习惯说法

倾向中 / 高必要性的线索:
- A–H 组高密度命中(尤其 C/G/H 结构信号)
- 句长均一、段落等大
- 通篇正确、安全、无个人风险判断
- 完美的三段对称、机械路标词

分流:
- **低:**没有明确问题,或只有单个弱信号。原文退回,只处理用户明确要求的格式 / 定点问题
- **中:**多个独立模式成簇出现,但结构和事实仍清楚。做 bounded 删改
- **高:**结构模板、内容注水与套话跨组共现,且能指出具体信息损失。允许结构重构

门检行必须给证据和置信度。不得写「AI 生成文本」或「真人文本」这类无法从文本证实的来源断言。

## 原有人味核对(阶段 4 用)

原文若包含以下内容,终稿应尽量保留:
- 一个具体的、带态度的判断(不是「这很重要」,而是「这步多半会砸,因为……」)
- 一处承认局限 / 不确定的诚实(「目前无法确认」「这条只能支持到这里」)
- 一个不讨好、但真实的观察

**关键约束**:不要求每篇都出现这类句子。原文没有就如实写「原文没有可保留的具体判断」,不得为了通过测试编造态度、经历或细节。

---

## 整篇五问(阶段 4 用)

终稿逐条过:
1. 原文已有的具体判断、感受或圈内说法是否保住了?原文没有时是否克制住了补写冲动?
2. 有没有把人味连同 AI 味一起删平?还认得出作者是个有判断力的人吗?
3. 所有数字 / 实体 / 引用是否和原文逐一一致?
4. 逐句删一遍,有没有「删掉也不损失信息」的空句?(空句检测)
5. 读起来是「懂行的人写给我看」,还是「AI 在输出格式化内容」?

**终极一问(借 best-humanizer-handbook 的「real reason to write it」):这稿读起来,像不像一个人有真实理由要写它,而不是被要求填一个格式?** 这比逐条规则更狠 —— 过了五问但仍像「为交付而交付」,说明人味还没真正注入。它是诊断,不是要你去制造戏剧性(对照 J 风格偏好里禁的假坦诚 / 伪洞察)。

任何一问不过,回阶段 2 / 3 对应处理。

---

## 空句检测(阶段 2 ＋ 五问第 4)

逐句问:「把这句整句删掉,读者损失了什么信息?」
- 损失了具体信息 / 判断 → 保留
- 只损失了「一种强调的感觉」「一句正确的废话」→ 标记为候选删除

典型空句:「这一点至关重要」「不可忽视其影响」「具有重要的意义」「总的来说还是不错的」。

---

## 独立子 agent 复核(高风险场景)

客户方案、特稿,或走「深」档时,派一个独立 agent(Claude Code 用 Task 工具;Codex 用其子 agent 机制),只给它原文 ＋ 终稿,让它回答:
1. 终稿相对原文,有没有任何数字 / 人名 / 机构 / 报价 / 引用被改动或丢失?逐项列。
2. 终稿读起来还像不像 AI?如果像,指出最像的两处。
3. 有没有过度净化 —— 原文的某处人味 / 判断被洗掉了?

主流程不自评自己;以独立 agent 的回答为准做最后一轮回改。
