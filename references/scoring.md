# 多轴评分细则

两个正交分 + 两类独立清单一起看:
- **AI 味分 0–100**:表达模式启发式,越低越干净,不用于判断作者身份
- **人味质量分 0–50**:越高越像人认真写,用于防止过度净化
- **个人偏好命中**:按真实命中计数,不并入 AI 味分
- **风险项**:事实 / 来源与格式 / 载体风险分级列出,不并入 AI 味分

只压 AI 味分容易洗成另一种机器腔。把个人偏好和风险混进 AI 味分,又会把「我不喜欢」或「格式不对」误报成 AI 来源证据。四轴必须分开。

## AI 味分(0–100,累加封顶)

按 `banned-words.md` 的 A–H 分组计分。I 组属于风险,J 组属于商业表达与个人风格,均不直接证明 AI 模式。

**先去重,后计分:**
- 同一文本片段同时命中多组时只取最高分,不得叠加
- 同一根因的连续命中合并成一个 pattern cluster
- 引用、专有名词、模板占位与合法行业术语不计分
- 200 字以下短文本置信度默认低,不得只凭单词命中判深档

**词汇 / 套话**
- B/D 组 Tier 1 强信号:每个去重 cluster **+6**,本类封顶 24
- B/D 组 Tier 2,同段第 2 次起:每处 **+4**
- E 组填充词:每处 **+3**;口播声口中承担呼吸、转场、反问或自我修正功能的软垫词不计入,只对无信息堆叠计分
- B 组 Tier 3:不逐处计,密度异常整体 **+5**

**结构(C/G/H,信号较强)**
- 机械三段式 / 格言公式 / 连续人造金句:每个 cluster **+8**
- 开场烘托句:**+8**
- 结尾感召段:**+8**
- 设问自答或假坦诚钩子:**+6**
- 装饰性结构化列表:**+6**
- 连续句节奏机械均质:**+6**
- 否定肯定并列滥用:**+5**

**内容 / 翻译腔(A/F)**
- 内容注水(同义反复 / 空转因果):每处 **+5**
- 翻译腔语序:每处 **+4**
- 普通英文词、物理动作比喻等弱信号只有在密度、语体不匹配和其它模式共现时才计分

累加超 100 按 100 计。0–15 支持轻档,16–35 支持中档。36 以上只有在至少命中一个结构问题且跨两个独立组时才支持深档;否则仍按中档处理。

## 个人偏好与风险项

- `user-taste.md` 与 J 组逐项列出真实引用,同一处只记一次
- **高风险:**数字、实体、引语或来源无法核对;疑似虚构引用
- **新增信息风险(改写时新引入的事实):** 改写中补进原文没有的数字、行业数据、案例、出处时,**一律单独标记并提示核实来源**,不得静默植入;即便是为增强说服力补的行业大盘数据,也要列入「新增信息」清单交用户核对(对齐 `user-taste.md`「数据必带口径 / 拿不准标待确认」)
- **中风险:**模板占位符、来源卡片、追踪参数、信息缺口
- **低风险:**目标载体不支持的 Markdown、空格、引号或标点格式
- 风险项只触发核对 / 标注 / 格式修复,不自动触发结构重写

## 计分注意

- 命中回上下文核验,不按裸字符串计数。「主打通勤」不命中「打通」,「商品卡」不命中「卡」
- 乙方 deck 的战略角色、KFS、节点日历、人群分层等专业结构不扣分;空话仍按信息损失判定
- 一个词既在 D 组又在 J 组时,AI 味最多记 D 组一次,J 组只出现在个人偏好清单
- 无法给出真实引用和命中理由时,不得给分
- 四字格密度、整数化框架(「三个维度 / 四个层面」)、三段排比按**文体阈值**判:品牌方案 / 事件营销 / 公文基线高于一般公众号;各项有实质差异、不可对调时不计分(见 `banned-words.md` C 组)
- **多层结构叠加不因单层合规而整体豁免**:几套独立的三 / 四分法(判断句数、段落标题、回避方式类型……)同时取同一整数时,即使每层单看都有实质差异、不可对调,叠加本身仍按 G 组「结构与节奏」计一次均质化分,处理方式是打散呈现节奏而非删结构(见 `banned-words.md` C 组「多层结构叠加」)
- 表格 / 脚本收尾的「为什么非用 X」论证句 ＋ 红线自查勾选行不计分,判据是各条是否指向具体、不可互换的核查点(见 `channel-presets.md` 乙方腔边界)
- 口语原生「但是 / 然后 / 所以」在口播 / 对话体不计入 B 组;承载真实因果时序的连接词不算 AI 高频词
- 内部对齐稿 / 方向稿语体下,工作化口语、括号备注、TODO 未完项不纳入 AI 味计分(是正确语体,不是 AI 腔残留)

## 人味质量分(0–50,五维各 0–10)

| 维度 | 满分长什么样 | 0 分长什么样 |
|------|------------|------------|
| **直接性** | 结论前置,不绕弯 | 层层铺垫才说重点 |
| **节奏** | 句子随内容自然变化 | 机械等长或连续短句造戏剧感 |
| **信任读者** | 点到为止,不过度解释 | 自问自答、反复强调 |
| **具体与保真** | 信息具体,判断有依据,没有擅自补事实 | 空泛或出现事实漂移 |
| **简洁** | 没有空句,每句带信息 | 大量填充与正确的废话 |

人味分 <25 即使 AI 味分很低,也说明洗过头了,触发反过度净化检查。

## 输出示例

```
【门检】改写必要性中,置信度高
【评分】AI 味 48→12 | 人味 28→41
【个人偏好】2 处:「不是 X,而是 Y」标题 ×1;「打透」×1
【风险项】中风险 1 处:原文数据没有来源,保留并标「待确认」
【证据】结构 cluster 2 个;D 组 Tier 1 cluster 1 个;同片段已去重
```

## 关于「降 AI 率 / 过检测」

本管线不以骗过 AI 检测器为目标。AI 味分是编辑启发式,只回答「哪些表达值得改」,不回答「谁写的」。
