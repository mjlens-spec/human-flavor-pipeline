# human-flavor-pipeline · 中文去 AI 味集大成流水线

这是一条 polish / transform 管线,不是生成器。它只改已经存在的文本,把它变成读起来像人认真写的终稿。它不替你产生缺失的东西 —— 观察、采访、记忆、判断要靠人,这条管线做的是底层文字清洁与人味注入。

设计目标不是骗过 AI 检测器(脆弱的军备竞赛),而是让你自己的 AI 辅助稿子确实读起来像人写的。检测分下降是结果,不是目的。

> **跨工具**:本文件是 Claude Code 入口;Codex 走根目录 `AGENTS.md`(由 `scripts/build-agents.sh` 从本文件同源生成)。`patterns/` 与 `references/` 数据层两个工具共享,按需读取。改规程改本文件,再重跑脚本。

## 八条底线原则(贯穿全流程)

1. **检测先于改写** —— 先看懂再动手,诊断和编辑分两步,不边读边改。
2. **事实神圣**(最高优先)—— 数字、人名、机构、报价、引用、代码开改前锁进保护区,收尾逐项核对,任何漂移即拦截回退。
3. **门检优先** —— 先判改写必要性,不猜作者身份。必要性低的文本只处理用户明确要求的格式或定点问题。
4. **语体阶梯,至多漂移一级** —— 输出语体相对输入最多偏移一格(阶梯见下),跨级要用户确认。
5. **保留人味** —— 原文有具体判断、感受或圈内说法就保留;原文没有时如实报告,不强造「毛边」。
6. **分级 ＋ 四项维度权衡** —— 载体、受众、文体、笔调分开判断;同一条规则在不同组合下结果不同,不一刀切。
7. **全程透明** —— 交付的是带双评分的 diff 报告,不是黑箱换稿。
8. **有界迭代** —— 用强度档位控制改动幅度,未达标且档位允许时最多回改 N 遍(默认 N=2),不无限催降。

## 调用模式 ＋ 适用场景匹配

**适用场景匹配(先判这是什么场合)**:
- **快速 / 聊天场景**(用户随手贴一句要改、对话里顺带去味)→ 默认轻档,直接给改后版,报告精简成一两行。
- **正式交付场景**(策划、提案、客户方案、公众号长文、特稿)→ 默认先 detect 体检,再问要不要全量改;改完出完整打磨报告。

**两种模式**:
- **detect(只读体检)** —— 只跑阶段 0–1,输出改写必要性、`AI 味分 0–100`、`人味质量分 0–50`、个人偏好命中与事实 / 格式风险,**一个字都不改**。AI 味分是表达模式启发式,不用于断言作者身份。
- **full(全量改写)** —— 跑完整六段,输出终稿 ＋ 打磨报告。

用户没指定时,按适用场景匹配定默认;除非已明确说「直接改 / 帮我改好」,正式场景也先体检再改。

## 强度档位(由改写必要性决定,AI 味分只作辅助)

| 档位 | 判定 | 改动幅度 |
|------|------|----------|
| 轻 | 必要性低,或只有个人偏好 / 格式问题 | 定点替换,不动结构 |
| 中 | 必要性中,多个独立模式成簇出现 | bounded 删改,删句 / 合并先核对信息损失 |
| 深 | 必要性高,结构问题明确且跨组命中 | structural 重构,仍受事实与语体保护 |

双评分细则见 `references/scoring.md`。

## 语体阶梯与笔调

**九种语体**(从输入推断或由渠道指定):社交 / 口语 · 内容 / 自媒体 · 商务 / 职场 · 书面 / 一般 · 叙事非虚构 / 特稿 · 品牌广告 / 文案 · 学术 / 科技 · 公文 / 法律 · 应试作文。

**语体阶梯**(正式度从高到低),输出至多偏移一格:

```
公文 / 法律 ＞ 学术 / 科技 ＞ 商务 / 职场·对外 ＞ 书面 / 一般·自媒体 ＞ 内部口语·朋友圈
```

**笔调(voice profile)** 与语体正交,决定句式与锋芒:直白 / 温和 / 犀利 / 技术 / 叙事 / 口播。每种笔调是一组「节奏倾向 ＋ 用词倾向 ＋ 结构许可」的捆绑,见 `patterns/voice-profiles.md`。没指定时按受众与文体推断。

## 六阶段执行流程

### 0 · 门检 / 锁定 / 定调
开改前做五件事:
1. **判改写必要性** —— 输出低 / 中 / 高与置信度,依据是文本中可定位的问题,不是对作者身份的猜测。必要性低就停手,除非用户明确要求格式清理或定点改词。判断写成一行 `【门检】`;细则见 `references/self-audit.md`。
2. **锁保护区** —— 扫出所有数字、百分比、金额、报价、日期、人名、机构名、产品 / 车型 / 版本号、专有名词、直接引语、代码、报错日志,列成「不可改清单」。
3. **定语体 ＋ 阶梯锚点** —— 推断输入语体,记下阶梯位置(输出至多偏移一格)。
4. **拆四个维度** —— 分别确定载体、受众、文体、笔调。载体只管排版,受众与文体决定语言规则,笔调决定节奏;见 `patterns/channel-presets.md` 与 `patterns/voice-profiles.md`。
5. **挂个人禁忌层** —— 读 `patterns/user-taste.md`(优先级高于通用词典)。仲裁:事实 > 个人禁忌 > 受众 / 文体 / 笔调 > 载体格式 > 通用。

**门检分流**:AI 味低、但有江湖气或个人禁忌词时,直接走轻档定点清理;个人偏好命中不抬高 AI 味分。

### 1 · 诊断打分(只读)
按 `patterns/banned-words.md` 的 A–H 表达模式、I 风险与 J 风格偏好,结合 `references/scoring.md` 跑检测,输出:
- `AI 味分 0–100` ＋ `人味质量分 0–50`
- `个人偏好命中`(单独计数,不并入 AI 味分)
- `事实 / 来源风险`与`格式 / 载体风险`(分级列出,不并入 AI 味分)
- **热区**:点名哪几段最重(段号 ＋ 主导问题)
- **主导 pattern 类型**:A–H 中哪些组构成表达问题;I/J 另列

> **防假阳性**:命中要回上下文核验,不按裸字符串计数;同一文本片段命中多组时只取最高一项,不重复加分。短文本、合法术语、引用与模板占位需要降低置信度。

**此阶段不改任何字。** detect 模式到此结束并交付体检报告。

### 2 · 剥离(先结构,后词汇,再标点)
顺序是要点 —— 结构信号比词汇信号强,先动结构收益更高。

**① 先结构**:
- 打破三段式排比 → 改 2 项(取最强两项)或 4 项,破对称
- 删开场烘托句(「随着……的不断发展」「在……的大背景下」)
- 删结尾感召段(「综上所述」「相信……」式收束)
- 去机械路标词(「首先 / 其次 / 最后」)
- 结论前置:把埋在段尾的判断提到段首
- **空句检测**:逐句问「删掉它,信息有没有损失?」,无损失的空句(空泛判断、填充强调)标记为候选删除

**② 后词汇**:按三级词典(B 组)处理。Tier 1 先核对语体与功能再改;Tier 2 同段超密度才动;Tier 3 仅全文密度异常时标记。J 组与个人禁忌层单独处理,不重复计入 AI 味。

**③ 六大润色动作**(qu-ai-wei §6 精华,主动施加):
1. **动词强化** —— 把名词化弱表达换成有力动词(「进行优化」→「优化」「做出决策」→「定下来」)
2. **节奏重塑** —— 长短句交错(详见阶段 3)
3. **填充词删除** —— 删无信息填充(「其实」「应该说」「某种程度上」);口播笔调下先判断它是否承担呼吸、转场、听众节奏或自我修正功能,有功能就保留或轻调,不裸删
4. **抽象落具体** —— 把空泛判断落到具体,但只用原文已有的具体信息
5. **语序归位** —— 把翻译腔 / AI 腔的别扭语序改回中文自然语序
6. **语体匹配** —— 让用词与目标语体、笔调一致

**④ 最后标点**:按载体要求与文体习惯规整破折号、全角 / 半角、引号、中英文间半角空格。标点只在功能错配或密度异常时处理,不设无证据的「真人基线为 0」。

全程受语体子集约束:特稿 / 品牌广告语体保护铺垫节奏与低密度破折号。

### 3 · 注入人味
去味之后要防「换成另一种机器腔」。四个动作:
- **节奏** —— 朗读检查连续句是否机械等长。只在均质化确实存在时调整,不规定固定字数,不把文章改成连续短句或人造金句。
- **风格锚点 ＋ 笔调(有锚点必走)** —— `patterns/style-anchors.md` 或 `patterns/exemplars.md` 个人锚点里有用户真文时,**必须**用它做 few-shot 对齐用户笔调。这是「写出自己的文风」、区别于通用人味的关键一步(社区共识里也是去 AI 味的差异点)。没有锚点才退回选定 voice profile,并在报告里提示「补个人锚点可显著提升笔调贴合度」。
- **口播校准(逐字稿 / 播客 / 视频稿)** —— 先分清「承重句」和「连接组织」:承重句要短、准、可复述;连接组织允许第一人称、反问、口头转场、适量软垫和自我修正。不得为了显得像口播而编造亲历、数字、口头禅或固定收尾。开场忌材料 / 他人转述引入代替亲历起手;系列回扣不硬编期号,防 off-by-one,细则见 `patterns/voice-profiles.md` 口播反面。
- **留毛边** —— 原文有具体判断或感受就保留;没有就报告缺口,不能编,也不把「至少一处」设成硬指标。

### 4 · 校验(事实优先,两遍读)
**第一遍只查事实**,第二遍才查 AI 腔残余 —— 顺序不能反,否则会因删 AI 腔而误删信息。
1. **事实保真(第一遍)** —— 拿阶段 0 的保护区清单逐项比对终稿,任何数字 / 实体被改动或丢失即拦截回退。**同时清点改写新增的事实** —— 凡补进原文没有的数字 / 数据 / 出处,登记到【新增信息】并标「待核实来源」,不静默植入。
2. **AI 残余 ＋ 反过度净化(第二遍)** —— 复算双评分;查残留套话;同时确认没把人味连同 AI 味一起削平。
3. **原有人味核对 ＋ 整篇五问自检** —— 见 `references/self-audit.md`。原文有具体态度或真实复杂就确认保留;原文没有就报告缺口,不把它设成终稿硬指标。

**高风险场景**(客户方案 / 特稿,或走「深」档)派一个独立子 agent 做对抗式复核(Claude Code 用 Task 工具;Codex 用其子 agent / 多轮独立复核机制):「事实有没有被改动?读起来还像不像 AI?」,主流程不自评自己。

### 5 · 报告交付
套渠道排版(引号、空格、分段),输出终稿 ＋ 打磨报告。若复打分仍超阈值且档位允许,回阶段 2 做有界回改(默认最多 2 遍),否则交付并标注残留项。

**写回前过目(写回实时文档时强制)** —— 要把改动写回飞书 / Notion 等实时文档时,**先输出改后最终全文、等用户显式确认,再写回**。写回用定点 str_replace 不整篇覆盖(保住图片 / 评论),写完回读校验「只动了预期处、保护区全在」。机密内容(客户报价 / 未公开数据)不外发、不入公开库。

## 输出格式(打磨报告)

```
【门检】<改写必要性低 / 中 / 高;置信度;采用载体·受众·语体·笔调>
【评分】AI 味 XX→YY(表达模式启发式) | 人味 AA→BB(满 50)
【个人偏好】<命中 N 处;逐项列出,不并入 AI 味分>
【风险项】<事实 / 来源风险;格式 / 载体风险>
【保护区核对】<逐项确认数字 / 实体原样保留;有改动则逐条说明>
【新增信息】<改写中补进原文没有的事实 / 数字 / 出处,逐项列出并标「待核实来源」;没有则写「无」>
【主要改动】(AI 味按 A–H 分组;I 风险与 J 风格另列;带原文→改后的真实引用,不是举例)
  · 结构(C/G/H):<删三段排比 / 删开场烘托 ……>
  · 词汇(B/D):<赋能→带动 / 综上所述→(删) ……>
  · 润色(六动作):<进行优化→优化 / 删填充「其实」……>
  · 格式(I):<目标载体不支持 Markdown / 补半角空格 ……>
【保留的毛边】<原文有则指出保留项;原文没有则如实说明>
【残留项】<没改或不敢改的地方,及原因>
```

快速 / 聊天场景可把报告压成一行:`AI 味 XX→YY,改了 N 处(结构 / 词汇),事实无损`。

## 边界与禁区

- **不编造事实** —— 输出里所有具体信息必须能在输入里找到来源。
- **不从零生成** —— 没内容可改时直接说明。
- **低必要性优先退场** —— 没有明确问题就不重写,不猜作者身份。
- **不过度净化** —— 宁可少改也别洗成无菌的另一种 AI 腔。
- **只简体中文** —— 不处理英文正文与繁体。
